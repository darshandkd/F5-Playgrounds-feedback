# F5 AI Playground — Feedback & Issue Tracker

This repository is the **public issue tracker** for the [F5 AI Playground](https://kind-beach-0b0b70c1e-2.westus2.7.azurestaticapps.net/) — a hands-on, interactive demonstration of F5's AI infrastructure capabilities.

> **Heads-up:** This repo intentionally contains **no source code**. The application source lives in a separate private repository.
> Everything here is for filing bugs, requesting features, and asking questions about the live experience.

---

## What the F5 AI Playground is

A single-page web app that makes F5's AI portfolio explorable in 60 seconds:

- **AI Playground** — 30+ interactive simulations across Inference (request routing, semantic caching, GPU utilization, RAG, MCP), Safety (prompt injection defense, AI red-teaming, DLP), Data Delivery, and 14 animated AI concept explainers
- **EOB Playground** — Edge-on-BIG-IP (Tawon EOB) labs covering eBPF kernel-level observability, 5G core topology, architectural comparisons, and protocol tracing
- **Built for** F5 sales engineers running customer demos, F5 employees onboarding to AI products, and anyone curious about how modern AI infrastructure actually works

**Live URL:** https://kind-beach-0b0b70c1e-2.westus2.7.azurestaticapps.net/

## How to use this repo

### Found a bug?
[Open a Bug Report](https://github.com/darshandkd/F5-AI-Playground-feedback/issues/new?template=bug_report.yml). Please include:
- Which playground (AI Playground, EOB Playground, or the Gallery landing)
- Which simulation, tab, or feature
- Browser + OS
- A screenshot if visual
- Steps to reproduce

### Have a feature idea?
[Suggest a Feature](https://github.com/darshandkd/F5-AI-Playground-feedback/issues/new?template=feature_request.yml). Tell us what you want to demonstrate or learn that the playground doesn't cover yet.

### Just have a question?
[Ask a Question](https://github.com/darshandkd/F5-AI-Playground-feedback/issues/new?template=question.yml) or check existing issues — yours might already be answered.

### Browsing existing reports
- [All open issues](https://github.com/darshandkd/F5-AI-Playground-feedback/issues)
- [Bugs](https://github.com/darshandkd/F5-AI-Playground-feedback/issues?q=label%3Abug+is%3Aopen)
- [Feature requests](https://github.com/darshandkd/F5-AI-Playground-feedback/issues?q=label%3Aenhancement+is%3Aopen)
- [Recently fixed](https://github.com/darshandkd/F5-AI-Playground-feedback/issues?q=label%3Afixed+is%3Aclosed)

## Triage & response

Maintained by **[@darshandkd](https://github.com/darshandkd)** (d.doshi@f5.com).

Typical response window: **2-3 business days**. Reproducible bugs with screenshots usually get triaged the fastest. Feature requests are evaluated quarterly against the demo roadmap.

When a fix ships, you'll see it on the live URL within ~5 minutes of the maintainer closing the issue.

## Why is this repo separate from the source code?

The playground is built and maintained by an internal F5 marketing/engineering function. The source is kept private to protect ongoing product roadmap work, but the experience itself is publicly accessible — and we want a public way to hear from the people using it.

If you're an F5 employee who needs source access, reach out to **d.doshi@f5.com** directly.
